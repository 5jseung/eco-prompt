## 1. 지금까지 누가 무엇을 했나

### 지승 (지원 작업 + 개발 어시스트)

`main` 브랜치 (초기 세팅) + `feat/animated-character` 브랜치
- **프로젝트 환경 셋업.** Vite + React + Tailwind 기반 단일 페이지 앱으로 구성. `npm install` 한 번이면 누구나 바로 실행할 수 있는 상태.
- **GitHub 저장소 생성 + 협업 워크플로우 세팅.** `main` 브랜치 보호 + feature 브랜치 → PR 머지 흐름. README에 모든 셋업 절차 포함.
- **8개 프롬프트 섹션 1차 구현.** 와이어프레임을 기준으로 PURPOSE / SUBJECT / STYLE / CONTEXT / LIGHTING / COMPOSITION / MOOD / NEGATIVE 섹션을 모두 폼으로 구성.
- **물범 캐릭터 1차 (정적 PNG 5컷) 우측 하단 배치.** 섹션을 채울수록 표정이 단계별로 바뀜.
- **Gemini API 연동 백엔드 (Vercel serverless).** API 키가 없을 때는 텍스트 fallback. (오철님이 더 발전시킴)
- **현진님이 만들어주신 캐릭터 영상 에셋 6종 (.mov)을 웹용으로 변환.**  
  → 알파 채널 유지하면서 APNG로 변환, 총 16MB로 압축, 우측 하단에 자동 재생.
  - `stage1~5.png` : 완성도에 따른 표정 변화
  - `final_celebration.png` : 섹션 완료 / 프롬프트 생성 시 축하 애니메이션

### 오철 (`feat/prompt-generation-only` 브랜치)

오철님이 한 차례 큰 리팩토링을 해주셨고, **이 브랜치가 현재의 메인 브랜치가 될 예정.**

- **이미지 생성 → 프롬프트 정제 모드로 전환.** Gemini가 직접 이미지를 만드는 게 아니라, 우리가 입력한 폼 데이터를 받아 **90~150단어짜리 매끄러운 영문 이미지 프롬프트로 다시 써줌.** 사용자는 이 프롬프트를 복사해서 본인이 원하는 이미지 생성기에 직접 붙여넣음.
  - 이건 회의록에서 얘기했던 "프로토타입은 최종 프롬프트를 텍스트로 출력 → 연구 진행 시 수동으로 Gemini에 넣기" 폴백 시나리오를 한 단계 더 끌어올린 것.
- **섹션 접기/펼치기 기능.** 프롬프트 생성 버튼을 누르면 8개 섹션이 자동으로 접히면서 선택한 값만 칩 형태로 요약돼서 보임. 다시 클릭하면 펼쳐서 수정 가능.
- **로컬 개발 환경에서 `/api/generate` 동작.** Vite 미들웨어를 추가해서 `npm run dev`만 띄워도 Gemini API 호출이 동작하도록 함 (전엔 `vercel dev` 필요했음).
- **README, .env.example, .gitignore 정비.**

### 현진 (디자인 / 에셋)

- 물범 캐릭터 일러스트 5컷 + 축하 애니메이션 1컷 제작 → 지승이 웹용으로 변환해서 적용 완료.
- 펭귄 / 꿀벌 캐릭터는 아직 미완성 (현재는 물범만 적용).

## 2. 현재 GitHub 브랜치 상태

```
main                              ← 초기 세팅만 있음 (아직 정비된 버전 아님)
└─ feat/prompt-generation-only    ← 오철님 작업 (프롬프트 정제 모드 + 접기)
   └─ feat/animated-character     ← 지승 작업 (영상 캐릭터)
```

→ 합치는 순서: `feat/prompt-generation-only` 를 먼저 `main`에 merge, 그 다음 `feat/animated-character`를 merge.

## 3. 클릭으로 따라하는 셋업 튜토리얼

### 처음 한 번만 하는 작업

**1) Git, Node, GitHub CLI 설치 확인**

터미널 (Mac은 Terminal.app, Windows는 PowerShell)을 열고:

```bash
git --version       # 2.x 이상 나오면 OK
node --version      # v18 이상 나오면 OK
gh --version        # GitHub CLI. 없으면: Mac → `brew install gh` / Windows → https://cli.github.com
```

**2) GitHub 로그인**

```bash
gh auth login
```
- 질문 답변 순서: `GitHub.com` → `HTTPS` → `Yes` → `Login with a web browser`
- 화면에 나오는 8자리 코드를 복사 → 엔터 누르면 브라우저 열림 → 코드 붙여넣고 로그인.

**3) 저장소 클론**

본인이 작업하고 싶은 폴더로 이동한 다음:

```bash
git clone https://github.com/5jseung/eco-prompt.git
cd eco-prompt
```

**4) 의존성 설치 (한 번만)**

```bash
npm install
```
이거 1분 정도 걸림. 다 끝나면 `node_modules` 폴더가 생김.

**5) (선택) Gemini API 키 설정**

폼 결과를 실제로 매끄러운 프롬프트로 정제해보고 싶으면:

1. https://aistudio.google.com/apikey 에서 키 발급
2. 프로젝트 폴더에 `.env` 파일 생성
3. 파일 안에 `GEMINI_API_KEY=받은_키_여기에` 한 줄 추가
4. **저장 후 절대 GitHub에 push 하지 말 것.** (`.gitignore`에 등록되어 있어서 자동으로 빠지지만 그래도 확인.)

키 설정 안 해도 앱은 동작함. 단지 Gemini 정제 단계 없이 우리가 만든 로컬 프롬프트 초안이 그대로 출력될 뿐.

### 매번 작업 시작할 때 하는 작업

**1) 최신 코드 받기**

```bash
git checkout main
git pull
```

**2) 작업할 브랜치로 이동**

지금은 오철님 작업 + 지승 캐릭터 작업이 별도 브랜치에 있으므로, 우선 둘 다 보고 싶으면:

```bash
# 오철님 버전 보기
git checkout feat/prompt-generation-only

# 지승 캐릭터 + 오철님 버전 둘 다 보기
git checkout feat/animated-character
```

**3) 개발 서버 실행**

```bash
npm run dev
```

자동으로 브라우저에서 http://localhost:5173 이 열림.  
멈추고 싶으면 터미널에서 `Ctrl + C`.

---

### 본인 작업 push 하는 법

**1) 본인 작업 브랜치 만들기**

```bash
git checkout -b feat/내-기능-이름
# 예: git checkout -b feat/reference-image-upload
```

**2) 파일 수정 → 저장**

코드 에디터(VS Code 추천)로 수정. 저장하면 `npm run dev`가 떠 있는 화면에 바로 반영됨.

**3) 커밋 + push**

```bash
git add .
git commit -m "feat: 기능 설명 한 줄"
git push -u origin feat/내-기능-이름
```

**4) Pull Request 만들기**

터미널 메시지에 뜨는 URL 클릭, 또는 https://github.com/5jseung/eco-prompt 가서 노란색 배너 "Compare & pull request" 클릭 → 설명 적고 "Create pull request" 클릭.

다른 팀원이 리뷰하고 머지하면 끝.

⚠️ **`main` 브랜치에 직접 push 하지 말 것.** 반드시 브랜치 만들고 PR로 머지.

## 4. 코드를 만지게 된다면 어디를 보면 되나

| 바꾸고 싶은 것                          | 파일                                   |
|--------------------------------------|--------------------------------------|
| 섹션 종류 / 옵션 추가 / 한국어 라벨 수정     | `src/data/sections.js`               |
| 프롬프트 조립 방식 (한국어 → 영어 변환 등)   | `src/utils/buildPrompt.js`           |
| 캐릭터 위치 / 애니메이션 트리거           | `src/components/Character.jsx`       |
| 페이지 전체 레이아웃 / 헤더 / 하단 출력 영역  | `src/App.jsx`                        |
| 섹션 요약 표시 방식                      | `src/components/Section.jsx`         |
| 색상 / 폰트 / 키프레임 애니메이션           | `tailwind.config.js`                 |
| Gemini 프롬프트 정제 지시문              | `api/generate.js` (상단 instruction)  |
| 펭귄 / 꿀벌 캐릭터 추가                  | `public/seal/anim/` 옆에 `public/penguin/`, `public/bee/` 폴더 만들고 같은 이름의 파일 넣기 |

## 6. 문제 생기면

- `npm install` 에러 → Node 버전이 v18 미만일 수 있음. https://nodejs.org/ 에서 LTS 버전 다운로드.
- `git push` 가 비밀번호를 물어보면 → `gh auth login` 다시 실행.
- 화면이 깨지거나 옛날 버전이 보이면 → 브라우저 강력새로고침 (Mac: `Cmd + Shift + R`, Windows: `Ctrl + Shift + R`).
- 어떤 브랜치에 있는지 모르겠으면 → `git branch` 입력. 별표(*)가 붙어 있는 게 현재 브랜치.
