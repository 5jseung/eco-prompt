# 지승 치트시트 — git + dev server

매번 이거 한 장만 보면 됩니다. 위에서 아래로 순서대로.

---

## 0. 매번 시작할 때 (작업 시작 전 1번)

```bash
cd ~/Desktop/26-1/hci      # 프로젝트 폴더
git checkout main          # main 브랜치로
git pull                   # 팀원이 올린 최신 변경 받기
```

> `git pull` 했을 때 빨간 글씨로 "conflict" 안 뜨면 OK.
> 떴으면 → 아래 **자주 만나는 에러** 참고.

---

## 1. 서버 켜기 (개발용)

```bash
npm install                # 처음 한 번 + package.json 바뀌었을 때
npm run dev                # 매번
```

브라우저에서 → http://localhost:5173 (포트는 콘솔에 뜨는 거 사용)

**끄는 법:** 터미널에서 `Ctrl + C`

> Gemini로 진짜 프롬프트 생성까지 보고 싶을 때만 `.env`에 `GEMINI_API_KEY=...` 채우면 됩니다.
> 안 채워도 텍스트 fallback 으로 동작은 합니다.

---

## 2. 새 작업 시작할 때 (브랜치 만들기)

```bash
git checkout main
git pull
git checkout -b feat/뭐를할지         # ← 이름은 영어로, 공백 ❌
```

브랜치 이름 예시:
- `feat/penguin-character` — 새 기능
- `fix/seal-hover-zindex` — 버그 수정
- `docs/update-readme` — 문서만 수정

---

## 3. 작업 중간/끝에 (저장하고 올리기)

```bash
git status                 # 뭐가 바뀌었는지 확인
git add -A                 # 모든 변경 스테이지
git commit -m "feat: 펭귄 캐릭터 추가"
git push -u origin feat/penguin-character   # 첫 push는 -u 필요
```

이후 같은 브랜치에서 더 push 할 때는 그냥 `git push` 만 하면 됩니다.

**커밋 메시지 컨벤션 (선택):**
- `feat:` 새 기능
- `fix:` 버그 수정
- `docs:` 문서만
- `chore:` 잡일 (deps, config 등)

---

## 4. PR 만들기 & 머지하기 (GitHub 웹)

1. push 한 직후 터미널에 보이는 URL 클릭 → "Create pull request"
2. 또는 https://github.com/5jseung/eco-prompt/pulls → "New pull request"
3. base: `main` ← compare: 내 브랜치
4. 제목/설명 짧게 적고 "Create pull request"
5. 오철님이 리뷰 → 초록 **Merge pull request** 버튼 → **Confirm merge**
6. 머지된 브랜치는 GitHub UI에서 "Delete branch" 눌러서 정리

**머지 후 내 컴퓨터로 동기화:**
```bash
git checkout main
git pull
git branch -d feat/penguin-character    # 로컬 브랜치 삭제
```

## 5. 자주 만나는 에러

### "Your branch is behind 'origin/main' by N commits"
```bash
git pull
```

### "CONFLICT" 가 떴다
충돌난 파일을 에디터로 열어서 `<<<<<<<` 표시 찾아 수동 정리 → 저장 후:
```bash
git add 충돌난파일
git commit                 # 메시지 자동
git push
```
혼자 못 풀겠으면 그대로 두고 → "도와줘 conflict 났어" 라고 저한테 말씀해 주세요.

### "Authentication failed" 가 떴다
git push 할 때 비밀번호 물어보고 거절되면:
```bash
gh auth login              # GitHub.com → HTTPS → Login with browser
```
(한 번만 하면 됨)

### `npm run dev` 가 갑자기 빨간 글씨로 멈춤
```bash
rm -rf node_modules package-lock.json
npm install
npm run dev
```

### 실수로 main 에 바로 commit 해버렸을 때
```bash
git branch feat/내가만든것      # 현재 commit을 새 브랜치에 저장
git reset --hard origin/main   # main을 깨끗하게 되돌림
git checkout feat/내가만든것    # 새 브랜치로 이동
git push -u origin feat/내가만든것
```

## 6. 모르겠을 때 (마법의 명령어)

```bash
git status                 # 지금 상태가 어떻지?
git log --oneline -10      # 최근 commit 10개
git branch -a              # 모든 브랜치 (로컬 + 원격)
```

이 3개 결과 복붙해서 저한테 보내주시면 진단 가능합니다.

---

## 7. 절대 하지 말 것

- ❌ `git push --force` (이거 치면 팀원 작업이 사라질 수 있어요)
- ❌ `.env`, `.env.local` 을 commit (Gemini 키가 GitHub 에 박힘)
- ❌ `main` 에서 바로 코드 수정 + commit (브랜치를 먼저 만들 것)
- ❌ `node_modules/` 를 commit (`.gitignore` 가 막아주지만 혹시)
